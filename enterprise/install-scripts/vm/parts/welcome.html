Welcome to OpenShift

App delivery just got a whole lot easier.

This virtual machine is pre-configured as an all-in-one deployment of OpenShift Enterprise by Red Hat, complete with the JBoss Developer Studio. From here you can develop applications directly against the world-class OpenShift Enterprise Platform-as-a-Service.

The Web Console

Click below to view the OpenShift Enterprise web console. You will need these login credentials:

  • Username: demo
  • Password: openshift
Open the Web Console

The rhc Utility

Open a terminal on this system (Applications - System Tools) and run the following command to get started with rhc, the command-line utility for developing apps with OpenShift Enterprise:

$ rhc setup --server localhost -l demo -p openshift

JBoss Developer Studio

Open JBDS on this system (Applications - Programming menu), click past the splash screen, and select File - New - OpenShift Application from the menu (even to import an existing application). You will need the account credentials:

  • Username: demo
  • Password: openshift

External access

This VM and applications created on it have DNS records that are only available from the BIND server contained in the VM. It is possible to resolve VM host and application names externally by adding the IP of this VM as the nameserver for the domain openshift.example.com. The method of doing this is specific to your operating system and environment; e.g. for Linux you would add this to the top of /etc/resolv.conf:

nameserver 192.168.x.x

More details are available here. The VM IP must also be reachable from the host (check with ping).

Be aware that the cerficate of the VM web server is self-signed, and should be expected to cause a certificate warning when visited by external browsers or tools.

Changing passwords

This host is pre-configured with a host login and OpenShift user for ease of use. The default passwords can be changed as follows.

You can change the password for the host "openshift" login from the terminal window with the passwd command.

$ passwd
(enter the current password, "openshift")
(enter your new password twice)

You can change the password for the OpenShift "demo" user (or add other logins) from the terminal window with the htpasswd command.

$ sudo htpasswd /etc/openshift/htpasswd demo
(enter your new password twice)
